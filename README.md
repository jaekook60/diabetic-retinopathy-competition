# Diabetic Retinopathy Classification Competition

안저(Fundus) 이미지를 이용해 **당뇨병성 망막병증(Diabetic Retinopathy)** 의 중증도 단계를 분류한 경진대회 정리 저장소입니다.  
본 저장소에는 대회에서 사용한 코드, 발표 자료 일부, 성능 결과 및 분석 내용을 정리했습니다.

---

## 1. 대회 개요

- **대회명**: [데이터 애널리틱스 과목 경진대회]
- **참가 형태**: 팀 참가
- **주제**: 안저 이미지 기반 당뇨병성 망막병증 단계 분류
- **성과**: **1위 / 수상**
- **역할**: 데이터 이해, 모델 학습, 성능 평가, 결과 분석, 발표 자료 정리 전 과정을 직접 수행

---

## 2. 문제 정의

당뇨병성 망막병증은 당뇨로 인해 망막의 혈관이 손상되어 시력 저하를 유발할 수 있는 대표적인 안과 질환입니다.  
조기에 발견하면 치료를 통해 시력 저하를 예방할 수 있기 때문에, 안저 이미지를 활용한 자동 분류 보조 시스템의 필요성이 큽니다.

본 경진대회에서는 안저 이미지를 바탕으로 질환의 중증도를 **0단계부터 4단계까지 분류**하는 것을 목표로 하였습니다.

---

## 3. 클래스 구성

본 대회에서는 총 5개 클래스로 분류를 수행하였습니다.

| 등급 | 설명 |
|------|------|
| 0단계 | 정상 |
| 1단계 | 경증 비증식성 - 미세혈관류만 존재 |
| 2단계 | 중등도 비증식성 - 출혈, 미세혈관 이상 일부 존재 |
| 3단계 | 중증 비증식성 - 다수의 출혈, 광범위한 혈관 폐쇄 |
| 4단계 | 증식성(PDR) - 신생혈관 형성 및 유리체 출혈 등 시력 손상 위험 |

### 클래스 분포
- 0단계: 5800
- 1단계: 100
- 2단계: 300
- 3단계: 50
- 4단계: 250

클래스 간 샘플 수 차이가 커서 **심한 클래스 불균형 문제**가 존재했습니다.

## 5. 접근 방법

본 경진대회는 안저 이미지를 입력으로 받아 당뇨병성 망막병증 단계를 분류하는 **의료 이미지 분류 문제**로 접근하였습니다.

### 사용 모델 및 방법
- Backbone: **ConvNeXt-Large**
- Validation Strategy: **Stratified 5-Fold**
- Evaluation: **OOF(Out-of-Fold) Classification Report**
- Metric: **Accuracy, Macro F1-score**
- Framework: **PyTorch, FastAI, timm**

### 설계 포인트
- 클래스 불균형이 심하므로 단순 accuracy만으로 성능을 판단하지 않음
- 소수 클래스 성능까지 함께 보기 위해 **Macro F1-score** 중심으로 해석
- fold별 클래스 비율을 유지하기 위해 **Stratified K-Fold** 적용
- 전체 학습 데이터에 대한 일반화 성능을 보기 위해 **OOF 평가** 진행

---

## 6. 결과

전체 학습 데이터에 대해 **OOF(Out-of-Fold) Classification Report** 기준으로 다음과 같은 성능을 확인했습니다.

### 주요 성능
- **Accuracy**: `0.9183`
- **Macro F1-score**: `0.5961`
- **Weighted F1-score**: `0.9197`

### 클래스별 성능

| Class | Precision | Recall | F1-score | Support |
|------:|----------:|-------:|---------:|--------:|
| 0 | 0.9704 | 0.9597 | 0.9650 | 5800 |
| 1 | 0.2927 | 0.2400 | 0.2637 | 100 |
| 2 | 0.4624 | 0.5333 | 0.4954 | 300 |
| 3 | 0.5185 | 0.5600 | 0.5385 | 50 |
| 4 | 0.6773 | 0.7640 | 0.7180 | 250 |

### 결과 해석
- 전체 Accuracy는 높게 나타났지만, 정상 클래스 비중이 매우 크기 때문에 accuracy만으로 성능을 판단하기에는 한계가 있습니다.
- 따라서 클래스별 성능을 고르게 반영하는 **Macro F1-score(0.5961)** 를 함께 확인했습니다.
- 특히 샘플 수가 적은 **1단계, 3단계 클래스에서 성능 저하**가 나타났으며, 이는 데이터 불균형의 영향을 반영합니다.
- 반면 **0단계와 4단계는 상대적으로 높은 성능**을 보였습니다.

## 10. 한계 및 개선 방향

이번 결과를 통해 다음과 같은 한계를 확인할 수 있었습니다.

- 클래스 불균형이 매우 심해 소수 클래스 성능이 낮게 나타남
- Accuracy는 높지만, 소수 클래스까지 고려한 성능은 충분하지 않음
- 단일 모델 기반 접근이므로 추가적인 앙상블 실험 여지가 있음
- 오분류 사례를 더 체계적으로 분석하면 모델 개선 방향을 구체화할 수 있음

향후에는 다음과 같은 방향으로 개선할 수 있습니다.

- 소수 클래스 증강 및 재샘플링
- Focal Loss / Weighted Loss 비교
- 다양한 backbone 성능 비교
- 앙상블 적용
- Grad-CAM 등 XAI 기반 해석 추가
